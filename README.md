# EPSS-TPG: Exploit Prediction with Text Property Graphs

Artifact branch: `artifact/epss-tpg`.

This repository holds the EPSS-TPG model, the pipeline that builds a
Text Property Graph (TPG) from a CVE input text, the multi-view GNN
that reads the graph, the hybrid graph-plus-tabular head, and the
batch scripts that reproduce the training and evaluation runs.

## What is intentionally excluded

The following live on disk locally for development but are not part
of the artifact branch:

| Path | Reason |
|---|---|
| `EPSS_TPG/tpg_chatbot/` | Separate chatbot project. |
| `EPSS_TPG/EPSS_Latex/` | LaTeX figures and write-ups (kept in a separate place). |
| `EPSS_TPG/tests/` | Unit tests excluded from artifact. |
| `EPSS_TPG/install-tl-*/` | TeX Live installer dropped during tooling setup. |
| `EPSS_TPG/logs/` | Runtime training logs. |
| Per-experiment graph caches in `EPSS_TPG/data/epss_*/pyg_dataset/` | Large derived caches; regenerated by the pipeline from the source CSVs. |
| Per-experiment predictions / training history under `EPSS_TPG/output/**/` | Large per-CVE intermediate files; only the small `test_results.json` summary per run is kept. |

See [`EPSS_TPG/.gitignore`](EPSS_TPG/.gitignore) for the full list.

## Datasets (submodules)

Two datasets are referenced by the training scripts and are pinned
as git submodules. They are intentionally separate repositories so
that the dataset version is reproducible and the model code stays
small.

| Submodule | Holds |
|---|---|
| `Sec4AI4Aec-EPSS-Enhanced` | The social-media dataset CSVs (GPT, Gemma, Mistral, DeepSeek LLM summaries per CVE). |
| `Sec4AI4Aec-EPSS-Enhanced-megavul` | The Megavul commit-based dataset CSVs (separate `megavul` branch). |

To clone the full artifact with both datasets:

```bash
git clone --recurse-submodules https://github.com/Affan47/Text_property_Graph.git
cd Text_property_Graph
git checkout artifact/epss-tpg
```

If the repository is already cloned without submodules:

```bash
git submodule update --init --recursive
```

The dataset CSVs are tracked via Git LFS in the submodules, so a
working `git-lfs` installation is required to pull them.

## Reproducing the experiments

Three classes of run are scripted:

1. **Social-media ablation (20 runs):** four LLMs (GPT, Gemma,
   Mistral, DeepSeek) crossed with five text-source variants
   (`D`, `S_all`, `S_git`, `S_cvss`, `ALL`).
2. **Megavul ablation (15 runs):** three LLMs (GPT, Gemma, Mistral)
   crossed with five variants (`D`, `S_url`, `S_code`, `S_cvss`,
   `ALL`).
3. **NVD/KEV reference runs:** binary KEV classification and the
   two temporal-shift configurations.

The same matrix is also run with `--no-security-frontend` to
ablate the security frontend (38 additional runs).

See [`EPSS_TPG/README.md`](EPSS_TPG/README.md) for the full
training, testing, and inference commands.
